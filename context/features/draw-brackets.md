# Feature: Draw & Bracket Generation

## Trigger

`POST /api/categories/{id}/draw`

```java
public record DrawRequest(
    BracketType bracketType,   // optional: SINGLE_ELIMINATION, REPECHAGE, ROUND_ROBIN, POOL
    Boolean shuffle,            // optional: randomize entry order (default: false = seed order)
    Boolean enableRepechage     // optional: override category.repechageEnabled
)
```

**Pre-conditions**:
- Category must not already have matches → `BusinessConflictException`
- Need ≥ 2 active entries (not WITHDRAWN/DISQUALIFIED) → `BusinessConflictException`

## Algorithm (DrawServiceImpl.draw())

### Step 1 — Load and sort active entries
```
entries = category entries WHERE status NOT IN [WITHDRAWN, DISQUALIFIED]
if shuffle: randomly reorder
else: sort by seedNo (nulls last), then createdAt ascending
```

### Step 2 — Calculate bracket dimensions
```
bracketSize = nextPowerOfTwo(entries.size)
  // 3 entries → 4, 5 entries → 8, 9 entries → 16
roundCount = log2(bracketSize)
  // bracketSize=8 → 3 rounds: QF, SF, F
```

### Step 3 — Determine bracket type
```
enableRepechage = request.enableRepechage() ?? category.repechageEnabled
bracketType = request.bracketType ?? (enableRepechage ? REPECHAGE : SINGLE_ELIMINATION)
if bracketType == POOL: reject 400 Bad Request
```

### Step 4 — Create Bracket entity
```
Bracket { category, type, size=bracketSize, status=GENERATED }
```

### Step 5 — Generate matches per round

For `SINGLE_ELIMINATION` and `REPECHAGE`:
```
For each round (0 to roundCount-1):
  matchesInRound = bracketSize / 2^(round+1)
  For each position (1 to matchesInRound):
    Create Match {
      tournament, category, bracket,
      matchNumber = sequential,
      roundName = BracketRuleEngine.roundName(roundCount, roundIndex),
      roundNumber = round+1,
      bracketPosition = position,
      status = SCHEDULED,
      mode = category.discipline
    }
    If kumite mode: create KumiteMatchState { default scores, timer=180000ms }
```

For `ROUND_ROBIN`, `DrawServiceImpl` creates one category/group where every active entry faces every other active entry once:
```
bracketSize = entries.size
roundCount = 1
for akaIndex in 0..entryCount-2:
  for aoIndex in akaIndex+1..entryCount-1:
    Create Match {
      matchNumber = sequential,
      roundName = "Round Robin",
      roundNumber = 1,
      bracketPosition = sequential pairing number,
      status = SCHEDULED,
      mode = category.discipline
    }
    Create MatchParticipant { match, entry=entries[akaIndex], side=AKA }
    Create MatchParticipant { match, entry=entries[aoIndex], side=AO }
    If kumite mode: create KumiteMatchState using category.matchDurationSeconds
```

Round-robin match count is `n * (n - 1) / 2` (3 entries -> 3 matches, 4 entries -> 6 matches). This minimal implementation does not calculate standings, ranking tie-breaks, team scoring aggregation, or advancement out of pools.

**Round names** (generated by `BracketRuleEngine`):
- Last round → `"Final"` (or `"Chung kết"`)
- Second to last → `"Semifinal"` (or `"Bán kết"`)
- Third to last → `"Quarterfinal"` (or `"Tứ kết"`)
- Earlier rounds → `"Round of 16"`, `"Round of 32"`, etc.

### Step 6 — Connect winner paths

Applies only to `SINGLE_ELIMINATION` and `REPECHAGE`. `ROUND_ROBIN` matches do not have winner/loser next-match pointers.
```
For each round (0 to roundCount-2):
  For each match[i] in current round:
    nextRoundMatch = round[i/2]           // every 2 matches feed 1 next-round match
    match.winnerNextMatch = nextRoundMatch
    match.winnerNextSide = i%2==0 ? AKA : AO
```

### Step 7 — Create bronze medal matches (Repechage only)
```
If repechage AND roundCount >= 2:
  Create 2 bronze matches: roundNumber = roundCount+1, roundName = "Bronze Medal"
  Link semifinals: semifinal[0].loserNextMatch = bronze[0], loserNextSide = AKA
                   semifinal[1].loserNextMatch = bronze[1], loserNextSide = AKA
```

### Step 8 — Distribute entries to first-round slots

Applies only to `SINGLE_ELIMINATION` and `REPECHAGE`.
```
firstRound matches = matches in round 1
For slot 0 to bracketSize-1:
  If slot < entryCount:
    match = firstRound[slot/2]
    side = slot%2==0 ? AKA : AO
    Create MatchParticipant { match, entry=entries[slot], side }
```

### Step 9 — Auto-advance BYE matches

Applies only to `SINGLE_ELIMINATION` and `REPECHAGE`. `ROUND_ROBIN` has no BYE auto-advance.
```
For each first-round match:
  If participants.size == 1 (only one entry, other slot is BYE):
    match.winnerEntry = that entry's entry
    match.winType = BYE
    match.status = COMPLETED
    Create MatchParticipant in winnerNextMatch with winner entry
    If nextMatch has 2 participants now: nextMatch.status = READY
```

### Step 10 — Finalize
```
category.status = "DRAWN"
Return DrawResponse { bracketId, bracketSize, entryCount, matches[] }
```

## Data Model After Draw

```
Category (status="DRAWN")
  └─ Bracket (type=REPECHAGE, size=8, status=GENERATED)
       └─ Matches (organized by roundNumber)
            ├─ Round 1: QF matches (4 matches for 8-bracket)
            │    MatchParticipant (AKA side) → Entry → Athlete
            │    MatchParticipant (AO side)  → Entry → Athlete
            │    winnerNextMatch → SF match
            ├─ Round 2: SF matches (2 matches)
            │    winnerNextMatch → Final
            │    loserNextMatch → Bronze
            ├─ Round 3: Final (1 match)
            └─ Round 4: Bronze matches (2 matches, repechage only)
```

## BracketType Values

| Type | Description |
|------|-------------|
| `SINGLE_ELIMINATION` | Loser is out; no bronze matches |
| `REPECHAGE` | Semifinal losers get bronze match |
| `ROUND_ROBIN` | Minimal category/group support in `DrawServiceImpl`: all pairings, AKA/AO participants, and Kumite state duration; no standings or advancement out of pools yet |
| `POOL` | Explicitly rejected by `DrawServiceImpl` with `400 Bad Request` until multi-pool sizing and advancement rules are implemented |

## TEAM_KUMITE Aggregate Boundary

Draw generation still creates a normal bracket match per team entry pairing. It does not yet pre-create the WKF team series as multiple regular bouts, and it does not assign `Match.teamMatchGroupId`.

When grouped TEAM_KUMITE bouts do exist, the backend aggregate resolver uses:
- `teamMatchGroupId` to find all bouts in the same team match
- `teamBoutOrder` to order regular bouts and generated extra bouts
- `teamExtraBout` to mark Article 12.5 tie-break bouts

`MatchServiceImpl` defers bracket advancement for grouped TEAM_KUMITE until `TeamKumiteResolutionService` resolves the aggregate by bout victories, total points, or extra bout.

**Note**: `TournamentDrawServiceImpl` still auto-generates tournament-wide single-elimination/repechage draws from category settings. It has no request body for selecting `ROUND_ROBIN` or `POOL`; use `POST /api/categories/{id}/draw` with `bracketType=ROUND_ROBIN` for the minimal round-robin flow.

## BracketStatus Values

`DRAFT` (pre-draw), `GENERATED` (after draw), `LOCKED` (after all matches completed)

## Side Enum

`AKA` (red corner) and `AO` (blue corner) — standard karate terminology.
- AKA = even bracket slots (0, 2, 4...)
- AO = odd bracket slots (1, 3, 5...)

## Regional Variants
No regional variants in algorithm. Round names may have Vietnamese translations via `BracketRuleEngine`.

## Medical Withdrawal Propagation

After draw, a WKF 2026 Kumite 10-second-rule medical outcome does not physically remove participants or entries. `MatchServiceImpl` marks the injured athlete's remaining Kumite entries in the tournament as `WITHDRAWN`, then resolves future scheduled/ready Kumite matches that already contain those entries:
- one withdrawn side + one continuing side -> continuing side wins by `TEN_SECOND_RULE` and normal winner advancement runs
- both sides withdrawn or no continuing side -> match status becomes `CANCELLED`
- Kata categories are not affected
